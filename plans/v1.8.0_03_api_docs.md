# Plan: API Documentation

## Objective

Generate API documentation for `fetchext.core` and other public modules to support Library Mode users.

## Current State

- `fetchext` can be used as a library (`import fetchext`).
- Documentation exists for CLI but not for the Python API.

## Proposed Changes

1. **Tooling**: Use `pdoc` (auto-generating API docs for Python).
2. **Configuration**:
    - Add `pdoc` to `requirements-dev.txt`.
    - Add `make docs-api` target to `Makefile`.
3. **Output**:
    - Generate HTML documentation in `docs/api/`.
    - Integrate with existing MkDocs site if possible, or link to it.

## Implementation Details

- `pip install pdoc`
- `pdoc -o docs/api src/fetchext`
- Update `mkdocs.yml` (if it exists) to include API docs or just host them as static files.

## Verification

- Run `make docs-api`.
- Check if `docs/api/index.html` is generated.
- Verify content covers `fetchext.core`.
