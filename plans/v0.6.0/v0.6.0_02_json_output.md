# Plan: JSON Output (v0.6.0)

## Goal

Add a `--json` flag to `search` and `inspect` commands to output machine-readable JSON instead of rich tables. This enables integration with tools like `jq`.

## Specification

- **CLI**: Add `--json` flag to `search` and `inspect` subparsers.
- **Core**: Ensure core functions (`search_extension`, `inspect_extension`) return data structures (list of dicts, or dict) instead of just printing.
- **Output**:
  - If `--json` is passed, print the returned data as JSON to stdout.
  - If not, print the rich table (existing behavior).

## Files to Modify

- `src/fetchext/cli.py`: Add arguments, handle JSON printing.
- `src/fetchext/core.py`: Update functions to return data.
- `src/fetchext/downloaders/firefox.py`: Update `search` to return results.
- `src/fetchext/inspector.py`: Update `inspect` to return manifest/analysis.

## Verification

- **Unit Tests**: Verify core functions return data.
- **CLI Tests**: Verify `--json` flag produces valid JSON output.
