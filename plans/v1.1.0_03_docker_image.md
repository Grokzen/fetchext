# Plan: Docker Image (v1.1.0)

## Goal

Provide a containerized version of `fetchext` for easy deployment and usage in CI/CD pipelines without manual Python setup.

## Strategy

### 1. Dockerfile

- Base image: `python:3.11-slim` (small, secure).
- Install `git` (if needed for some hooks, but maybe keep it minimal).
- Install `fetchext` from source or PyPI.
- Entrypoint: `fext`.

### 2. GitHub Actions

- Create `.github/workflows/docker.yml`.
- Trigger on release (tag) or push to master.
- Build and push to `ghcr.io/grok/fetchext`.

### 3. Documentation

- Add "Docker Usage" section to `README.md` and `docs/installation.md`.

## Steps

1. Create `Dockerfile`.
2. Create `.github/workflows/docker.yml`.
3. Update docs.
