# Plan: Library Mode (v0.6.0)

## Goal

Decouple the CLI logic from the core functionality to make `fetchext` import-friendly. This allows other Python scripts to use `fetchext` as a library.

## Specification

- **Refactoring**: Move business logic from `src/fetchext/cli.py` into a new `src/fetchext/core.py` (or appropriate modules).
- **Public API**: Expose key classes and functions in `src/fetchext/__init__.py`.
- **CLI Wrapper**: `cli.py` should only handle argument parsing and logging setup, then call the core functions.

## Files to Modify

- `src/fetchext/cli.py`: Remove business logic, keep `argparse`.
- `src/fetchext/core.py`: Create this file (or `facade.py`) to hold high-level functions like `download_extension`, `batch_download`, `search_extension`.
- `src/fetchext/__init__.py`: Export public API.

## Verification

- **Unit Tests**: Ensure core functions can be called without `argparse` objects.
- **CLI Tests**: Ensure the CLI still works as expected (regression testing).
- **Script Test**: Create a small script `examples/library_usage.py` to demonstrate importing and using the library.
