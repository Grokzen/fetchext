# Plan: Plugin Manager

## Goal

Provide a CLI interface to manage Python-based plugins (hooks), allowing users to easily list, install, enable, and disable them.

## Requirements

1. **Command**: `fext plugin <subcommand>`
    * `list`: Show all plugins in `~/.config/fext/hooks`.
    * `enable <name>`: Enable a plugin (rename `name.py.disabled` -> `name.py`).
    * `disable <name>`: Disable a plugin (rename `name.py` -> `name.py.disabled`).
    * `install <path>`: Copy a python file to the hooks directory.
    * `remove <name>`: Delete a plugin file.
2. **Implementation**:
    * Update `src/fetchext/hooks.py` (or create `src/fetchext/plugins.py`) with `PluginManager` class.
    * `PluginManager` handles file operations.
3. **Integration**:
    * Add `plugin` subcommand to `cli.py`.

## Implementation Steps

1. **Create `src/fetchext/plugins.py`**:
    * `PluginManager` class.
    * Methods: `list_plugins`, `enable_plugin`, `disable_plugin`, `install_plugin`, `remove_plugin`.
2. **Update `src/fetchext/cli.py`**:
    * Add `plugin` subcommand.
3. **Tests**:
    * Unit tests for `PluginManager` using `pyfakefs`.

## Notes

* Plugins are just Python scripts in the hooks directory.
* "Enabled" means the file ends with `.py`.
* "Disabled" means the file ends with `.py.disabled`.
