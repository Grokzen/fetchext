# Plan: Logging Standardization (v1.5.0 Item 5)

## Goal

Ensure consistent logging practices across the codebase. Use `rich` for structured and colored logging. Standardize log levels (INFO for user-facing progress, DEBUG for internals, ERROR for failures).

## Current State

- `cli.py` configures `RichHandler`.
- Most modules use `logging.getLogger(__name__)` or `logging.getLogger("fetchext")`.
- Some modules might be printing directly to console instead of logging (besides the TUI/CLI output).

## Strategy

1. **Audit**: Check how loggers are initialized and used.
2. **Standardize**: Ensure all modules use `logging.getLogger(__name__)` or a shared logger.
3. **Levels**: Verify that `logger.info` is used for high-level progress, and `logger.debug` for details.
4. **Formatting**: Ensure messages are clear and consistent.
